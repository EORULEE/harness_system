# claim-evidence-rules.md — 주장-근거 감사 규칙 (AC10)

> 모든 Writing Suite workflow에 연결되는 공통 감사 규칙. `harness-claim-evidence-audit`
> 가 구현하고, 각 writer skill이 산출 직후 이 단계를 호출한다.

## 1. 원칙
- 모든 **사실 주장(claim)**은 **근거(evidence)**를 가져야 한다. 근거 없으면 "[확인 필요]".
- 날조 금지: 없는 인용·수치·기관명·실적·DOI를 지어내지 않는다(글로벌 feedback_references).
- 수치는 출처가 분명할 때만. 일반론("보통 ~일 것") 금지.

## 2. 감사 절차
1. 초안에서 사실 주장·수치·인용을 **추출**(목록화).
2. 각 항목에 근거 유형 태깅: `사용자제공` / `조사출처(DOI/URL)` / `계산` / `미확인`.
3. `미확인` 항목 → 본문에 **[확인 필요]** 마킹 + 사용자에게 1줄 보고.
4. 인용은 **실재 검증**(kb 라이브러리 등). 미검증 인용은 "출처 미확보"로.

## 3. 출력
- claim-evidence 표(주장 | 근거유형 | 출처 | 상태).
- 미확인/미확보 카운트. 0이 아니면 사용자 게이트(승인 전 확정 금지).

## 4. 연결 의무
- 각 writer SKILL.md는 워크플로 단계에 **본 감사 호출을 명시**해야 한다(AC10 검증 대상).
