논문에서 하네스가 지키는 원칙 (자동)

  • 인용 날조 금지 — 확인되지 않은 참고문헌은 쓰지 않고 "출처 미확보"로 표시
  • 근거 없는 단정 차단 — verification-gate 가 "…라서 불가능/원인은 X" 류 무근거 단정을 검사 (docs/VGATE.md)
  • 초안 전용 — 원고 파일을 직접 덮어쓰지 않음. 반영은 항상 사람이 결정
  • 결과 정직 — 성능 수치 비교는 여러 지표로, 좋은 것만 골라 쓰지 않음

포함되지 않은 것 (본인 계정으로 추가 가능 — ACCOUNTS.md)

reference manager(Zotero 등)·노트앱 연동·논문 검색 MCP 는 개인 계정이 필요해 코어에 미포함 — ACCOUNTS.md §MCP 절차로 본인 키로 붙이면 "관련 논문 찾아줘"까지 확장됩니다. 저널 심사위원 스타일 정밀 리뷰(8차원)는 공개 코어 미포함.

🧯 문제 해결 FAQ

1. "코드를 리뷰해줘"라고 했는데 수정이 됐어요?

"리뷰해줘"는 읽기 전용, "고쳐줘"는 수정 모드(B)입니다. 의도와 다르면 source diff 확인해줘라고 하고, 문제면 HOLD/rollback을 요청하세요.

2. Claude만 응답하고 Codex가 안 왔어요?

그 프로젝트의 Connector/라우팅이 아직 활성이 아닐 수 있습니다. 응답 끝의 처리 줄에 SINGLE_PLATFORM_PROVISIONAL이 보이면 임시 단일 결과입니다. "이 프로젝트 dual routing 상태 확인해줘"라고 물어보세요.

3. Zotero가 안 돼요?

Zotero 인증은 머신별 1회입니다. euru가 canonical이고 다른 머신은 Gateway 읽기 전용입니다. "이 머신 Zotero 인증 상태 확인해줘"라고 하세요. 한 source가 미인증이어도 나머지 기능은 정상입니다.

4. 다른 서버에서는 안 돼요?
