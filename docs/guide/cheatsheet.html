🗣️ 한 장 요청 치트시트

아래 문장을 그대로(또는 비슷하게) 말하면 됩니다. 라벨은 선택입니다.

파일·상태

이 파일이 어디 있는지 확인해줘.
현재 하네스 상태를 확인해줘.

코드

코드를 리뷰해줘.
이 코드의 버그를 찾아줘.
이 버그를 테스트와 함께 고쳐줘.
이 코드를 더 빠르고 안전하게 개선해줘.
이 코드와 논문의 Methods가 일치하는지 확인해줘.

데이터

이 CSV를 정리해줘.
결측치와 이상치를 확인해줘.
통계를 계산하고 그래프를 만들어줘.
학습 데이터와 테스트 데이터가 겹치는지 확인해줘.
원본은 수정하지 말고 결과를 새 폴더에 만들어줘.

딥러닝·실험

이 모델 학습 코드를 검토해줘.
실험 로그를 표로 정리해줘.
실제 학습은 하지 말고 실험 계획만 만들어줘.
실제 GPU 학습을 실행할 준비가 됐는지 검토해줘.

⚠ 실제 학습 실행은 C/V3 + 사용자 승인이 필요합니다.

논문·보고서

이 논문 리뷰해줘.
이 자료로 논문 초안을 작성해줘.
Methods와 Results를 작성해줘.
Reviewer 답변서를 만들어줘.
이 논문을 최종 제출 전에 검증해줘.
이 결과를 연구보고서로 정리해줘.

발표·시각화

이 결과로 발표자료를 만들어줘.
표와 그림을 논문용과 발표용으로 각각 만들어줘.
이 슬라이드를 더 이해하기 쉽게 고쳐줘.

Reference·지식

위키에서 이 방법의 근거를 찾아줘.
옵시디언에서 관련 결정과 노트를 찾아줘.
Zotero에서 이 주제의 문헌을 찾아줘.
이 내용을 옵시디언 정리 후보로 만들어줘.
이 결과를 Wiki 승격 후보로 만들어줘.

읽기는 기본 read-only. canonical write/승격은 별도 승인.

Git·환경 (Claude Code 기본 기능)

이 변경사항 커밋해줘.
이 작업 브랜치 만들어서 PR 올려줘.
의존성 설치하고 테스트 돌려줘.
빌드가 되는지 확인해줘.

커밋·PR·외부 push는 사용자가 요청할 때만 수행합니다.

릴리스·배포

이 릴리스가 배포 가능한지 검토해줘.
이 프로젝트를 서버에 배포해줘.
배포 후 문제가 없는지 확인해줘.

⚠ 고위험 — V3·백업·canary·사용자 승인 필요.

Prefix 예시 15+

A0 V0: 이 파일이 어디 있는지 확인해줘.
A0: 현재 상태 확인해줘.
A1 V1: 이 코드를 빠르게 검토해줘.
A1 V2: 이 코드를 교차검증해줘.
A1 V2: 이 논문 리뷰해줘.
A1 V3: 최종 제출 전 검증해줘.
A1: 이 설계 분석해줘.
B V1: 이 CSV 정리해줘.
B V2: 이 데이터를 처리하고 결과를 검토해줘.
B V2: 이 버그를 테스트와 함께 고쳐줘.
B: 이 함수 구현해줘.
C V3: 실제 실험 실행 계획을 검증하고 승인 요청해줘.
V2: 이 결과를 교차검증해줘.
[A1][V2] 이 표 수치 검증해줘.
A1/V2: 코드-논문 정합 확인해줘.
A1, V2: 방법론 검토해줘.
